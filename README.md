# Causal ML for Football Analytics: Yellow Card Analysis

Replication code for the empirical study reported in the paper *"Causal Machine
Learning for Football Analytics: Estimating Counterfactual Effects from
Observational Data"*. The paper estimates the causal effect of an early yellow
card on a player's defensive behaviour in the opening of the second half, using
StatsBomb open event data and double machine learning (DML).

## Repository structure

```
.
├── 01_fetch_statsbomb_events.ipynb   download competitions, matches, events
├── 02_build_analysis_frame.ipynb     construct the per-player-match analysis frame
├── 03_linear_regression.ipynb        naive OLS and OLS+W baselines (Table 2)
├── 04_dml.ipynb                      partial-linear DML: ATE + subgroup CATEs
├── 05_assumption_checks.ipynb        positivity, SUTVA, sensitivity analysis
├── src/
│   ├── analysis_config.py            single source of truth for W and Z
│   ├── build_subgroup_figure.py      generates the subgroup-ATE bar-grid figure
│   └── build_table2_numbers.py       prints the three-estimator comparison table
├── requirements.txt
└── README.md
```

Folders that are **not** tracked by Git:

- `data/` — fetched StatsBomb data and the derived `analysis_frame.csv`.
- `documents/` — figures generated by `build_subgroup_figure.py`, the DAG
  source/render (`dag.drawio`, `dag.pdf`), and the project slides.

## Reproducing the analysis

### 1. Install dependencies

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Fetch the data

```bash
jupyter nbconvert --to notebook --execute 01_fetch_statsbomb_events.ipynb --inplace
```

Downloads the StatsBomb open event data via `statsbombpy`. The fetch step may
take ~30 minutes depending on your network. Outputs land in `data/`.

### 3. Build the analysis frame

```bash
jupyter nbconvert --to notebook --execute 02_build_analysis_frame.ipynb --inplace
```

Produces `data/analysis_frame.csv` with one row per eligible player-match.

### 4. Run the analyses

```bash
jupyter nbconvert --to notebook --execute 03_linear_regression.ipynb --inplace
jupyter nbconvert --to notebook --execute 04_dml.ipynb --inplace
jupyter nbconvert --to notebook --execute 05_assumption_checks.ipynb --inplace
```

### 5. Regenerate the paper figure and Table 2 numbers

Both scripts read from `data/` and (for the figure) write to `documents/`. Run
them from the project root:

```bash
python src/build_subgroup_figure.py   # writes documents/fig_subgroup_*.png
python src/build_table2_numbers.py    # prints the comparison table to stdout
```

## Notes

- The DML pipeline uses `GroupKFold` on `match_id` for cross-fitting and
  cluster-robust standard errors at the match level, to respect within-fixture
  dependence.
